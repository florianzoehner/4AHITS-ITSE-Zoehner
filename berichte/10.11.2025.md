## Übung(port probing)

### Mit telnet prüfen, ob auf Metasploitable ein SSH-Server läuft

In Kali das Terminal öffnen und folgenden Befehl eingeben
telnet IP von Meta 22 -> 
telnet 192.168.5.51 22

Wenn eine Verbindung hergestellt wird, läuft ssh
Dann zeigt ssh einen Banner: SSH-2.0-OpenSSH_4.7p1 Debian-8ubuntu1

### Bezeichnung + Version des Webservers herausfinden (Port 80)

#### mit telnet
Befehl: telnet 192.168.5.51 80

Wenn Verbindung steht eine kleine HTTP Request eingeben:
HEAD / HTTP/1.0
Host: <IP>

Ausgabe: Server: Apache/2.2.8 (Ubuntu) DAV/2


#### mit netcat
Hier verwendet man ein Here-Document:

nc <IP> 80 <<EOF
HEAD / HTTP/1.0
Host: <IP>

EOF

Ausgabe ist dieselbe wie bei telnet.

#### mit curl
Befehl:curl -I http://192.168.5.51/

Ausgabe:
HTTP/1.1 200 OK
Date: Mon, 20 Oct 2025 12:17:49 GMT
Server: Apache/2.2.8 (Ubuntu) DAV/2
X-Powered-By: PHP/5.2.4-2ubuntu5.10
Content-Type: text/html



#### Cheetsheet
Telnet
telnet <IP> 22                       # SSH testen
telnet <IP> 80                       # HTTP Banner anzeigen
HEAD / HTTP/1.0
Host: <IP>


Netcat (nc)
nc -zv <IP> 1-30                     # Ports 1–30 scannen
nc -vz <IP> <PORT>                   # Einzelport testen
nc <IP> 80 <<EOF                     # HTTP Header abrufen
HEAD / HTTP/1.0
Host: <IP>

EOF

Curl
curl -I http://<IP>/                 # HTTP Header
curl http://<IP>/                    # Webseite abrufen

## Übung (netcat port scan)

Befehl: nc -zv 192.168.5.51 1-30


Die Ports die "Connected" sind:

| Port | Dienst | Typische Bedeutung      |
| ---- | ------ | ----------------------- |
| 21   | FTP    | Dateiübertragung        |
| 22   | SSH    | Remote Shell            |
| 23   | Telnet | Remote Shell (unsicher) |
| 25   | SMTP   | Mailserver              |


## Übung (nmap port scan)

#### Normaler Scan

Befehl: nmap 192.168.5.51
Der normale scan prüft die 1000 häufigten Ports.
![](berichte/bilder/nmap1.jpg)

#### Alle Ports scannen

Befehl: nmap -p- 192.168.5.51
Prüft alle Ports.

#### Service + Versions-Erkennung

Befehl: nmap -sV 192.168.5.51
Nmap probiert Banner + Fingerprinting und zeigt Versionen

#### OS Detection

Befehl: nmap -O 192.168.5.51

#### Timing/Speed Optionen

nmap -T4 192.168.5.51
nmap -T5 192.168.5.51

T4 = schnell und stabil
T5 = sehr schnell, aber unzuverlässig

#### Ausgabe in Dateien

nmap -oN scan.txt 192.168.5.51
nmap -oX scan.xml 192.168.5.51

nmap -oN scan.txt 192.168.5.51:

#### Option --reason

nmap --reason 192.168.5.51

#### Nur die n häufigsten Ports

nmap --top-ports 50 <IP>
→ deutlich schneller, ideal für erste Orientierung.


### Was kann man mit einem Portscan herausfinden?

- Ein Portscan kann zeigen:

- Welche Ports offen sind

- Welche Dienste laufen

- Versionsnummern der Dienste

- Betriebssystemindizien

- Netzwerkfilter/Firewalls

- Auffälligkeiten (z. B. ICS-Ports, RPC, veraltete Server)

## Übung (nmap cheat sheet)

| Tool            | Zweck                   | Befehl                                                           |
| --------------- | ----------------------- | ---------------------------------------------------------------- |
| **telnet**      | Port testen (z. B. SSH) | `telnet <IP> 22`                                                 |
| **telnet**      | HTTP-Header auslesen    | `telnet <IP> 80` +<br>`HEAD / HTTP/1.0`<br>`Host: <IP>`<br><br>  |
| **netcat (nc)** | Portscan 1–30           | `nc -zv <IP> 1-30`                                               |
| **netcat (nc)** | Einzelnen Port prüfen   | `nc -vz <IP> <PORT>`                                             |
| **netcat (nc)** | HTTP-Header abfragen    | `nc <IP> 80 <<EOF`<br>`HEAD / HTTP/1.0`<br>`Host: <IP>`<br>`EOF` |
| **curl**        | HTTP-Header anzeigen    | `curl -I http://<IP>/`                                           |
| **curl**        | Webseite abrufen        | `curl http://<IP>/`                                              |
