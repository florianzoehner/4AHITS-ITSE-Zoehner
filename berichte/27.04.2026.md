## Übung 1 – Analyse und Symbolic Mode

### Aufgabe 1 – Permissions auslesen

```bash
ls -l /tmp/lab/wrestle_plans.txt
# -rw-r--r-- 1 root root 6 ...
```

**User Owner:** root, **Group Owner:** root, **Permissions:** 644.  
Dateien bekommen bei `chmod -R 755` effektiv 644, weil das x-Bit bei normalen Dateien nicht gesetzt wird.

---

### Aufgabe 2 – Private Files

```bash
touch notes.txt
chmod o-r notes.txt   # 644 → 640
```

---

### Aufgabe 3 – Group Sharing

```bash
chmod u=rwx,g=rw,o= notes.txt
```

`o=` ohne Angabe entzieht *others* alle Rechte. Alle drei Klassen werden in einem Aufruf gesetzt.


### Aufgabe 5 – Kollektive Änderung

## Übung 2 – Ownership und Gruppenkollaboration

### Aufgabe 1 – sudoers Eintrag für blondie

Zuerst habe ich mit `sudo su` eine Root-Shell geöffnet und dann mit `visudo` die Datei `/etc/sudoers` bearbeitet:

```bash
sudo su
visudo
```

Folgenden Eintrag habe ich hinzugefügt:
blondie ALL=(root) /usr/bin/mv, /usr/bin/chown

Das erlaubt blondie, genau diese zwei Befehle (`mv` und `chown`) als root auszuführen – aber nichts anderes.

---

### Aufgabe 2 – Group Change

```bash
su - blondie
touch ~/song.txt
sudo cp ~/song.txt /tmp/lab/song.txt
sudo chgrp music /tmp/lab/song.txt
```

Funktioniert, weil blondie selbst Mitglied in *music* ist.

---

### Aufgabe 3 – su vs. su -

```bash
su blondie      # Wechsel ohne Login-Shell, Environment bleibt
su - blondie    # Vollständige Login-Shell, Home + Profile werden neu geladen
```

Mit `exit` kehrt man zum vorherigen User zurück.

---

### Aufgabe 4 – User Transfer

```bash
chown elvis /tmp/lab/song.txt
```

`chown` ohne `:` ändert nur den User Owner. Nur root darf das.

### Aufgabe 5 – Wrestler-Szenario

```bash
su - hogan
touch /tmp/lab/training.txt
chgrp wrestle /tmp/lab/training.txt
chmod g=rw,o= /tmp/lab/training.txt
```

Da *hogan* in *wrestle* ist, konnte er die Gruppe ohne `sudo` setzen.

### Aufgabe 6 – Einschränkungen

```bash
chgrp emperors /tmp/lab/song.txt
# → Operation not permitted
```

Ein User darf die Gruppe nur auf eine Gruppe setzen, der er selbst angehört. *elvis* ist nicht in *emperors* → Befehl schlägt fehl.

## Übung 3 – Directory Permissions und Octal Notation

### Aufgabe 1 – Oktale Umwandlung (r=4, w=2, x=1)

```bash
chmod 755 datei   # rwxr-xr-x
chmod 660 datei   # rw-rw----
chmod 700 datei   # rwx------
```

---

### Aufgabe 2 – Pub-Verzeichnis

```bash
mkdir /home/kali/pub
chmod 751 /home/kali/pub
# u=rwx(7), g=r-x(5), o=--x(1)
```

Das x-Bit auf Verzeichnissen ist die *Search-Permission* – es erlaubt `cd`, aber nicht `ls`.

---

### Aufgabe 3 – Recursive Lockdown

```bash
chmod -R g=rw,o= /tmp/lab/drafts/
```

 Für Verzeichnisse fehlt dadurch das x-Bit. `g=rwx,o=` wäre für Verzeichnisse sinnvoller.

---

### Aufgabe 4 – Search Permission

```bash
chmod o-r /home/kali
```

Andere User können `/home/kali` nicht mehr auflisten, aber dank x-Bit noch mit `cd` in bekannte Unterverzeichnisse wechseln.
