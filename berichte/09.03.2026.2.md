## Übung (ssh auf Metasploitable)

Als Vorbereitung für die folgende Übung wird ein **passwortloses SSH Login** auf der Metasploitable Maschine eingerichtet.  
Dabei wird ein **Public/Private Key Paar** verwendet.

Der Benutzer auf Metasploitable ist:

msfadmin


### 1. SSH Schlüssel auf Kali erstellen

Auf Kali wird zuerst ein RSA-Schlüssel erzeugt.

``` ssh-keygen -t rsa ```
Der Schlüssel wird in folgendem Verzeichnis gespeichert:

~/.ssh/id_rsa
~/.ssh/id_rsa.pub

id_rsa → privater Schlüssel

id_rsa.pub → öffentlicher Schlüssel


### 2. Public Key auf Metasploitable kopieren

Der öffentliche Schlüssel muss auf Metasploitable in folgende Datei eingefügt werden:

~/.ssh/authorized_keys

Dies kann  mit folgendem Befehl erfolgen:

ssh-copy-id -i ~/.ssh/id_rsa.pub msfadmin@192.168.5.51


### 3. Verbindung testen

Danach habe ich getestet, ob die Verbindung funktioniert:

ssh msfadmin@192.168.5.51

Da der SSH Server auf Metasploitable sehr alt ist, musste ich teilweise folgende Option verwenden:

``` ssh -oHostKeyAlgorithms=+ssh-rsa msfadmin@192.168.5.51 ```

Die Verbindung hat danach funktioniert und ich konnte mich ohne Passwort als msfadmin einloggen.

Die Datei authorized_keys ist wichtig, weil sie später in der nächsten Übung manipuliert wird.
